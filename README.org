* quantdb
A database schema and ingestion layer for storing arbitrary quantitative
and categorical measurements with full provenance tracking.

See the [[file:./docs/api.org][REST API documentation]] for details on
the external query interface.

** Ingestion Layer

The =quantdb.generic_ingest= module provides a zero-boilerplate API for
inserting data into the PostgreSQL schema.  FK columns accept
human-readable strings or dicts; the engine resolves them to integer/UUID
PKs automatically by walking the schema's FK graph.

Key components:

- =SchemaGraph= — runtime FK introspection via SQLAlchemy automap
  (topological sort, natural key detection, lookup table classification).
- =Ingest= — user-facing API with =row()=, =batch()=, =get()=, and
  =session()= methods.

*** Usage Example

#+begin_src python
from sqlalchemy import create_engine
from quantdb.models import reflect_models
from quantdb.generic_ingest import Ingest
from quantdb.utils import dbUri

engine = create_engine(dbUri('quantdb-user', 'localhost', 5432, 'quantdb'))
models = reflect_models(engine=engine)
ing = Ingest(models)

with ing.session() as s:
    # FK columns resolve automatically: "um" → units.id, "count" → descriptors_quant.id
    row = ing.row(s, 'values_quant',
                  value=42.0,
                  value_blob=42.0,
                  object='<object-uuid>',
                  desc_inst='nerve',
                  desc_quant='count',
                  instance={'dataset': '<dataset-uuid>', 'id_formal': 'sub'})

    # Batch insert shares an FK cache across rows
    results = ing.batch(s, 'values_quant', [
        {'value': i, 'value_blob': i, 'object': '...', 'desc_inst': 'nerve',
         'desc_quant': 'count', 'instance': {'dataset': '...', 'id_formal': 'sub'}}
        for i in range(10)
    ])

    # Look up existing rows
    unit = ing.get(s, 'units', label='um')
#+end_src
